# i18n Reference

Source files: `src/i18n/ui.ts`, `src/i18n/utils.ts`

Supported locales: `en` (default), `fr`

---

## `ui.ts`

Flat key-value map for all user-visible strings. Exported as the `ui` object.

```typescript
export type Lang = keyof typeof ui;          // 'en' | 'fr'
export const defaultLang: Lang = 'en';

export const ui = {
  en: { 'key': 'value', ... },
  fr: { 'key': 'value', ... },
} as const;
```

**Key conventions:**

| Pattern | Example |
|---------|---------|
| `<page>.<section>.<name>` | `frontis.hero.subtitle` |
| `<component>.<name>` | `nav.pelote` |
| `<page>.meta.title` | `index.meta.title` |
| `<page>.meta.description` | `kancha.meta.description` |

**Fallback behavior:** If a key exists in `en` but not in `fr`, `useTranslations` returns the
English value. Missing keys from both locales return the key string itself.

## `utils.ts`

### `getLangFromUrl(url: URL): Lang`

Extracts the locale segment from a URL pathname.

```typescript
getLangFromUrl(new URL('https://example.com/fr/frontis'))
// Returns: 'fr'

getLangFromUrl(new URL('https://example.com/frontis'))
// Returns: 'en'  (defaultLang)
```

**Parameters:**

| Param | Type | Description |
|-------|------|-------------|
| `url` | `URL` | Full URL object |

**Returns:** `Lang` — one of the keys of `ui` (`'en'` or `'fr'`), defaults to `defaultLang`

---

### `useTranslations(lang: Lang): (key: keyof ui['en']) => string`

Returns a typed translation function bound to the given locale.

```typescript
const t = useTranslations('fr');
t('nav.pelote')          // → French value, or English fallback
t('nonexistent.key')     // → 'nonexistent.key' (key itself)
```

**Parameters:**

| Param | Type | Description |
|-------|------|-------------|
| `lang` | `Lang` | Locale identifier |

**Returns:** Function `t(key)` — accepts only keys present in `ui['en']` (TypeScript enforced)

---

### `useTranslatedPath(lang: Lang): (path: string) => string`

Returns a path-prefixing function for the given locale.

```typescript
const tp = useTranslatedPath('fr');
tp('/frontis')    // → '/fr/frontis'
tp('/')           // → '/fr/'

const tp2 = useTranslatedPath('en');
tp2('/frontis')   // → '/frontis'   (no prefix for default locale)
```

**Parameters:**

| Param | Type | Description |
|-------|------|-------------|
| `lang` | `Lang` | Locale identifier |

**Returns:** Function `translatePath(path: string): string`

**Note:** The default locale (`en`) uses no path prefix, per `prefixDefaultLocale: false` in
`astro.config.mjs`.

---

## Astro config

**File:** `astro.config.mjs`

```javascript
i18n: {
  defaultLocale: 'en',
  locales: ['en', 'fr'],
  routing: {
    prefixDefaultLocale: false,
  },
}
```

| Option | Value | Effect |
|--------|-------|--------|
| `defaultLocale` | `'en'` | English routes have no prefix |
| `locales` | `['en', 'fr']` | Only these two locales are valid |
| `prefixDefaultLocale` | `false` | `/frontis` (not `/en/frontis`) for English |

---

## Locale resolution in components

All components resolve the locale from `Astro.currentLocale`:

```astro
---
const lang = (Astro.currentLocale ?? 'en') as Lang;
const t = useTranslations(lang);
const tp = useTranslatedPath(lang);
---
```

`Astro.currentLocale` is `undefined` outside a locale-prefixed route, so `?? 'en'` ensures a
valid fallback.
